New to these tools? Quick glossary of terms you'll see on this page
CLI = Command Line Interface. A text-based way to talk to your computer (Terminal app on Mac).
Claude Code / Codex = AI assistants that live in your terminal and can read, write, and run code on your behalf.
Plugin = An add-on that gives your AI assistant new capabilities (like apps on a phone).
Skill = A reusable workflow template an AI agent can follow — like a standard operating procedure for AI.
MCP (Model Context Protocol) = A standard that lets AI tools connect to external services (browsers, databases, APIs). Think of it as USB for AI.
Remote Control / Dispatch = Send tasks to your AI from your phone while it works on your laptop.
Compound Engineering = A methodology where every task you complete makes the next task easier — your AI gets smarter over time.
Obsidian = A note-taking app that stores everything as plain text files — perfect for AI to read and write.
CONTEXT.md / STRATEGY.md = Plain text files that tell your AI about your business, products, and decisions so it doesn't start from scratch every session.
Vercel = A platform that puts your websites and tools on the internet with one command.

First time using Terminal? Start here.

Terminal is a built-in Mac app that lets you type commands to your computer. Everything in this setup guide uses Terminal.

How to open Terminal:

1

Press Cmd + Space to open Spotlight search

2

Type Terminal and press Enter

3

A window with a text prompt appears — this is where you'll paste the commands from this guide

How to run a command: Copy the command text (use the Copy button next to each code block), paste it into Terminal (Cmd + V), and press Enter. Wait for it to finish before running the next one.

If something goes wrong: Don't panic. Copy the error message and bring it to your setup call with Jack. Most errors are one-line fixes.

6Foundational Frameworks

These diagrams illustrate the core concepts and methodologies we'll cover in the workshop. Study them before the session — we'll build real tools using each framework.

Five Foundations for Working With AI

The habits that make AI more valuable the more you use it — regardless of what you're working on.

For Amazon sellers: This is the operating system for how you'll work with AI. Gather context (competitor data, market research), compress it into knowledge your AI remembers, plan and execute in small verified steps, save what you learn so next time is faster, then turn winning processes into reusable skills that run automatically across all your products.

The Shared Context Stack

How persistent files make every AI session smarter than the last — so you never re-explain your business to AI again.

For Amazon sellers: Imagine your AI already knows your brand story, your target customer, your pricing strategy, why you chose that supplier, and every lesson from past launches. That's what these files do — they persist between sessions so your AI never starts from zero.

The Agentic CI/CD Pipeline

The full loop from "I have an idea" to "it's live and verified" — every stage AI-augmented.

For Amazon sellers: This is how you go from "I want to improve my listing" to "it's live, tested, and I can roll back if something breaks." Spec your idea, let AI plan and implement it, automatically test and review, deploy with a safety net, and capture what you learned so next time is even faster.

AI-Native Project Management

How to give your AI agents a structured to-do list instead of free-form chat — so they know what to work on, what's done, and what's next.

For Amazon sellers: Instead of telling AI "optimize my listings" in a chat that forgets everything, you create structured tasks: "Audit ASIN X for keyword gaps," "Pull competitor pricing for category Y." The AI picks up tasks, completes them, and the results are tracked — like having a VA that never loses the thread.

Domain-Driven Design for AI Agents

Six proven patterns for managing complexity — originally developed for large software systems, now applied to AI agent workflows.

For Amazon sellers: The #1 reason AI gives bad answers is ambiguous language. Does "launch" mean first day of inventory or first day of PPC? Does "profitability" include FBA fees? These patterns give you and your AI a shared vocabulary — so it stops guessing and starts getting it right.

Troubleshooting

"brew: command not found"?
Homebrew isn't installed. Run: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" then restart your terminal.
Can't install Claude Code or Codex?
Make sure you have Node.js 18+ installed: node --version. Install with brew install node or from nodejs.org.
Claude Code authentication failing?
claude auth logout && claude auth login
Codex authentication failing?
Run codex auth again. Ensure you're on the Pro ($100/mo) plan.
Plugin or skill not showing up?
Restart Claude Code or Codex after installing. Plugins and skills load at session start only.
Everlearn ZIP will not unzip?
Make sure the file is in ~/Downloads, then run cd ~/Downloads && unzip -o everlearn-workshop.zip. If your browser renamed it, use the exact filename shown in Finder.
Everlearn note not visible in Obsidian?
Open Obsidian and choose Open folder as vault, then select ~/Everlearn Vault. The test note should be inside 30 Research/Customer Signals.
Playwright MCP not working?
Ensure Node.js 18+ (Node 16 gives "performance is not defined" error). On Linux: npx playwright install-deps
Claude in Chrome not connecting?
Must be Google Chrome or Microsoft Edge. Brave, Arc, and other Chromium browsers are not supported.
Computer Use not available in Codex?
Requires the Codex Desktop app (not CLI) and is macOS only as of May 2026.
