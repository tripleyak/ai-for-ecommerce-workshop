New to these tools? Quick glossary of terms you'll see on this page
CLI = Command Line Interface. A text-based way to talk to your computer. You can use Apple's Terminal app or Ghostty.
Claude Code / Codex = AI assistants that can run in desktop apps or terminal CLIs and can read, write, and run code on your behalf.
Plugin = An add-on that gives your AI assistant new capabilities (like apps on a phone).
Skill = A reusable workflow template an AI agent can follow — like a standard operating procedure for AI.
MCP (Model Context Protocol) = A standard that lets AI tools connect to external services (browsers, databases, APIs). Think of it as USB for AI.
Remote Control / Dispatch = Send tasks to your AI from your phone while it works on your laptop.
Compound Engineering = A methodology where every task you complete makes the next task easier — your AI gets smarter over time.
Obsidian = A note-taking app that stores everything as plain text files — perfect for AI to read and write.
CONTEXT.md / STRATEGY.md = Plain text files that tell your AI about your business, products, and decisions so it doesn't start from scratch every session.
Vercel = A platform that puts your websites and tools on the internet with one command.

7Autoresearch Lab: Build A Self-Improving Forecast Tool

This module shows you how to set up a local recursive self-improvement loop on your MacBook. The examples are ProfitCurve-style response curves and demand forecasting, but the pattern works for any measurable business workflow. ~30 minutes

What you are building. A local autoresearch workspace with a fixed evaluation harness, one editable model file, sample ecommerce data, an experiment log, and an agent prompt that lets Codex or Claude Code improve the model over many runs.
Operating-system fit: this lab maps to the Product Intent to Verified Output loop. You define the business decision, fix the scorecard, let AI run controlled experiments, validate the result, then capture the improved workflow.

1. Two Starter Tracks

TrackInput dataWhat the loop improves
Response curvesfamily,date,spend,salesRecreate the core ProfitCurve idea: fit spend-to-sales curves and reduce holdout WMAPE.
Demand forecastingseries,date,units,price,ad_spend,market_signal,promoBuild a forecasting loop that improves future unit-demand predictions.

2. Workflow And Benchmarks

Step 1
Fix The Scorecard
Keep prepare.js, sample data, holdout split, and WMAPE metric stable.
Step 2
Edit One File
The agent changes only train.js, so the model improves without moving the goalposts.
Step 3
Run The Harness
run-experiment.sh scores the attempt and appends a row to results.tsv.
Step 4
Keep Or Discard
Better WMAPE gets kept. Worse WMAPE gets rolled back inside the starter repo.
Step 5
Package The Pattern
The same loop becomes a reusable workflow for other measurable ecommerce tasks.

4. The Autoresearch Pattern

PiecePurposeRule
program.mdDefines the MacBook-local autoresearch workflow.Read first.
<track>/program.mdDefines the track goal, metric, and allowed ideas.Read second.
prepare.jsLoads data, splits holdout, and scores WMAPE.Do not edit.
train.jsContains the model or fitting logic.This is the only file the agent edits.
results.tsvLogs every experiment, metric, status, and description.Append after each run.
scripts/setup-run.shCreates an autoresearch/<track> branch and baseline result.Run once per track.
scripts/run-experiment.shRuns the fixed scorecard and appends the result.Run after each idea.
GitKeeps good experiments and discards bad ones.Only inside the starter folder.

6. Agent Prompt

Run this from the starter kit root after choosing a track.

Claude / Codex
Read program.md and the selected track program.md. Start or continue an autoresearch loop.

Rules:
- Only edit train.js.
- Do not edit prepare.js or data files.
- After each idea, commit the attempted train.js change.
- Run ./scripts/run-experiment.sh response-curves pending "short description"
  or ./scripts/run-experiment.sh demand-forecast pending "short description".
- Keep the change only if holdout WMAPE improves.
- If it gets worse, discard the train.js change and try a new idea.
- Prefer simple changes that improve the metric.

7. What The Agent Will Try

1

Run the baseline model and read the current WMAPE.

2

Modify the editable model file with one experiment idea.

3

Rerun the fixed harness against the chronological holdout set.

4

Keep the change if WMAPE improves; discard it if the metric gets worse.

5

Repeat until the loop finds a better response-curve or demand-forecasting method.

Troubleshooting

"brew: command not found"?
Homebrew isn't installed. Run: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" then restart your terminal.
Can't install Claude Code or Codex CLI?
Make sure you have Node.js 18+ installed: node --version. Install with brew install node or from nodejs.org. If the CLI path is blocked, use the Claude Desktop or Codex app path and bring the error to setup help.
Claude Desktop Code tab missing or stuck?
Update Claude Desktop from claude.ai/download, restart the app, then sign out and back in. If you use the CLI, run claude doctor from Ghostty.
Codex app or CLI authentication failing?
Open the Codex app and sign in with your ChatGPT account. For the CLI, run codex again and follow the browser sign-in prompt. Confirm your plan includes Codex access.
Plugin or skill not showing up?
Restart Claude Desktop, Claude Code, or Codex after installing. For Claude Desktop app-level skills, check Customize > Skills. For Claude Code local skills, check ~/.claude/skills/. For Codex skills, check the local skill folder used by the installer and reload skills from the app if available.
Ghostty installed but the command does not work?
Open Ghostty from Applications or Spotlight. Some macOS app installs do not add a ghostty command to your shell path. That is fine for the workshop.
Computer Use cannot see or click apps?
Open macOS System Settings > Privacy & Security and check Screen Recording and Accessibility permissions for Claude or Codex. Restart the app after changing permissions.
Everlearn ZIP will not unzip?
Make sure the file is in ~/Downloads, then run cd ~/Downloads && unzip -o everlearn-workshop.zip. If your browser renamed it, use the exact filename shown in Finder.
Everlearn note not visible in Obsidian?
Open Obsidian and choose Open folder as vault, then select ~/Everlearn Vault. The test note should be inside 30 Research/Customer Signals.
Playwright MCP not working?
Ensure Node.js 18+ (Node 16 gives "performance is not defined" error). On Linux: npx playwright install-deps
Claude in Chrome not connecting?
Must be Google Chrome or Microsoft Edge. Brave, Arc, and other Chromium browsers are not supported.
Computer Use not available in Codex?
Requires the Codex app (not CLI) and is macOS only as of May 2026.
